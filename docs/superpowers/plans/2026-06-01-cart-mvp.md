# Cart MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 Java 后端实现第一版购物车能力，让登录用户可以把已有 SKU 加入购物车、查询、修改数量和删除，为后续订单与支付链路提供稳定前置数据。

**Architecture:** 本阶段继续采用模块化单体。新增 `cart` 模块，使用 Spring Web MVC 暴露 `/api/cart/**` 普通用户接口，使用 Service 层保护当前用户边界并通过 MyBatis XML 访问 `cart_item` 表。购物车只保存用户意图，不锁库存、不扣库存、不生成价格快照。

**Tech Stack:** Java 21, Spring Boot 4.0.6, Spring Web MVC, Spring Security OAuth2 Resource Server, MyBatis XML, Flyway, MySQL 8.0, H2 test profile, JUnit 5, MockMvc.

## 1. 阶段边界

### 本阶段实现

- 新增 `cart_item` 表。
- 新增 Java `cart` 模块。
- 当前登录用户可以添加 SKU 到购物车。
- 当前登录用户可以查询自己的购物车。
- 当前登录用户可以修改购物车项数量。
- 当前登录用户可以删除单个购物车项。
- 当前登录用户可以清空自己的购物车。
- 同一个用户重复添加同一个 SKU 时合并数量。
- 所有购物车接口都必须使用 `Authorization: Bearer <accessToken>`。
- 所有查询和写入都必须绑定当前 `userId`，禁止访问或修改其他用户购物车。
- 使用 MyBatis XML 做数据访问。
- 使用 Mapper、Service、Controller 测试覆盖核心行为。
- 更新 `backend-feature-mapping.md` 和 `api-testing-with-reqable.md`。

### 本阶段不实现

- 不创建订单。
- 不扣减库存。
- 不锁定库存。
- 不释放库存。
- 不做支付。
- 不做优惠券。
- 不做购物车选中状态。
- 不做购物车价格快照。
- 不做购物车商品上下架快照。
- 不做 Python internal cart API。
- 不做 AI 代客加购。
- 不做 SSE / WebSocket。
- 不做 MQ 异步任务。

### 设计原则

- 购物车是用户购买意图，不是交易凭证。价格、库存、上下架状态以后在创建订单时重新校验。
- Controller 只负责鉴权用户解析和入参校验；当前用户隔离必须在 Service 和 SQL 条件中同时体现。
- `cart_item` 使用 `(user_id, sku_id)` 唯一约束，重复添加通过 Service 合并数量。
- 第一版数量必须为正整数，建议最大值限制为 `99`，避免误操作和异常大数量污染数据。
- 删除和修改时必须按 `user_id + sku_id` 操作，不能只按客户端传入的单字段操作。

## 3. 数据库设计

### `cart_item`

```sql
CREATE TABLE cart_item (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    sku_id BIGINT NOT NULL,
    quantity INT NOT NULL,
    created_at DATETIME(6) NOT NULL DEFAULT CURRENT_TIMESTAMP(6),
    updated_at DATETIME(6) NOT NULL DEFAULT CURRENT_TIMESTAMP(6),
    UNIQUE KEY uk_cart_item_user_sku (user_id, sku_id),
    KEY idx_cart_item_user_updated (user_id, updated_at),
    CONSTRAINT fk_cart_item_user FOREIGN KEY (user_id) REFERENCES user_account(id) ON DELETE CASCADE,
    CONSTRAINT fk_cart_item_sku FOREIGN KEY (sku_id) REFERENCES product_sku(id),
    CONSTRAINT ck_cart_item_quantity_positive CHECK (quantity > 0)
);
```

说明：

- `user_id + sku_id` 唯一，避免同一用户同一 SKU 出现多行。
- 第一版不保存价格，订单创建时从商品库重新读取价格。
- 第一版不保存库存，订单创建时从库存表重新校验并锁定或扣减。
- `ON DELETE CASCADE` 只用于用户删除时清理购物车；SKU 不级联删除，避免历史购物车静默丢失。

## 4. API 契约

### 4.1 查询购物车

```http
GET /api/cart/items
Authorization: Bearer <accessToken>
```

响应：

```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "skuId": 2004,
      "spuId": 1002,
      "spuCode": "JACKET_COMMUTE_001",
      "name": "通勤轻薄外套",
      "category": "外套",
      "color": "黑色",
      "size": "L",
      "salePrice": 299.0,
      "stockStatus": "in_stock",
      "mainImageUrl": "/images/products/jacket-commute-main.svg",
      "quantity": 2
    }
  ],
  "errorCode": null,
  "message": "ok"
}
```

### 4.2 添加购物车项

```http
POST /api/cart/items
Authorization: Bearer <accessToken>
Content-Type: application/json
```

请求：

```json
{
  "skuId": 2004,
  "quantity": 1
}
```

行为：

- 如果当前用户购物车里没有该 SKU，插入新行。
- 如果已有该 SKU，累加数量。
- 如果累加后数量超过 `99`，返回参数错误。
- 如果 `skuId` 不存在，返回资源不存在。

### 4.3 修改购物车项数量

```http
PUT /api/cart/items/{skuId}
Authorization: Bearer <accessToken>
Content-Type: application/json
```

请求：

```json
{
  "quantity": 3
}
```

行为：

- 只能修改当前用户自己的购物车项。
- `quantity` 必须在 `1..99`。
- `skuId` 不存在于当前用户购物车时返回资源不存在。

### 4.4 删除单个购物车项

```http
DELETE /api/cart/items/{skuId}
Authorization: Bearer <accessToken>
```

行为：

- 只能删除当前用户自己的购物车项。
- `skuId` 不存在于当前用户购物车时返回资源不存在。

### 4.5 清空购物车

```http
DELETE /api/cart/items
Authorization: Bearer <accessToken>
```

行为：

- 删除当前用户全部购物车项。
- 不影响其他用户。
- 购物车已空时也返回成功。

购物车完成后，订单阶段可以基于购物车继续设计：
