# 衣橱画像浅色主题修复设计

## 问题

`ProfilePreferencesPage` 仍使用旧版 `noir` 深色页面类，但它现在嵌在新版浅色个人中心中。全局样式同时保留旧版的 `--text`、`--muted`、`--surface-raised`，并由新版令牌覆盖 `--surface`，导致深色卡片、浅色输入框、浅色输入文字和深色标题混用，文字与背景对比度不足。

## 目标

- 衣橱画像页面统一为个人中心的浅色米白视觉。
- 标题、标签、输入内容和控件边界清晰可辨。
- 保持现有字段、保存行为、数据转换、接口调用、测试标识和响应式布局不变。

## 方案

采用页面级浅色样式作用域：从 `ProfilePreferencesPage` 移除旧版 `noir` 主题类，增加专用的浅色页面类，并在商城样式中定义该页面的容器、卡片、标题、标签、表单控件、聚焦态和操作按钮。

不通过全局重定义旧版变量修复，因为那会影响登录、聊天等仍依赖深色主题的页面；也不只叠加针对 `noir` 的临时覆盖，以免继续保留错误的主题语义。

## 视觉规则

- 页面容器：米白色背景、浅棕边框、适度圆角。
- 内容卡片：接近白色的暖色表面、浅棕边框、轻量阴影。
- 主标题与卡片标题：深森林绿。
- 辅助标签：中深灰绿色，保证浅色背景上的可读性。
- 输入框、选择框和文本域：白色背景、深色文字、清晰边框。
- 聚焦态：绿色边框与半透明绿色外圈。
- 保存按钮：商城主绿色背景、白色文字；禁用态仍可辨认。
- 响应式行为：沿用现有双栏到单栏的断点。

## 影响范围

- `frontend/src/pages/ProfilePreferencesPage.tsx`：仅调整页面主题类名。
- `frontend/src/styles/commerce.css`：增加衣橱画像的浅色作用域样式。
- 如需回归保护，测试只验证稳定的主题类契约，不测试具体像素值。

## 验证

1. 运行个人画像页面组件测试。
2. 运行全部前端单元测试。
3. 运行生产构建。
4. 使用本地模拟数据打开衣橱画像页面，检查标题、标签、已有输入值、空输入框、选择框、日期框、文本域、聚焦态和窄屏布局。

## 非目标

- 不修改用户资料、身体数据或偏好接口。
- 不重构全站旧版深色样式。
- 不改变字段排列、文案或保存交互。
