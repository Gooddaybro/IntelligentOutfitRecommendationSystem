# NOIR.AI 前端视觉进化设计

## 1. 目标与边界

本轮将现有 React + Vite 前端从浅色“AI 导购工作台”统一升级为 **NOIR.AI 深色未来奢华体验**。视觉应具有电影感、冷光玻璃质感和明确的 AI 策展感，但避免霓虹滥用、复杂 3D 与会影响商品阅读的装饰。

### 目标

- 登录成功后以短暂、轻量、可跳过的分层动画进入工作台。
- AI 导购页以“左侧对话，右侧推荐”为核心结构；右侧采用“AI 首选大卡 + 辅助推荐卡”的策展主舞台。
- 登录、AI 导购、商品中心、购物车、订单和偏好页切换为同一套深色未来视觉系统。
- 强化推荐结果、AI 匹配度、推荐理由、商品图片与购买动作的视觉优先级。
- 保持桌面端项目展示的冲击力，同时确保窄屏可用。

### 不在范围内

- 不新增售后、退款、支付、订单或商品推荐业务能力。
- 不改变 Java 后端、Python 服务、共享契约、接口字段或请求时机。
- 不改变登录、聊天、流式回复、推荐、加购、立即购买、购物车、订单、偏好等既有业务行为。
- 不引入 Canvas、WebGL、3D 引擎或额外 UI 组件库。

## 2. 已确认的体验决策

| 决策 | 结论 |
| --- | --- |
| 整体风格 | 未来奢华工作台：深色基调、冷色渐变、玻璃表面、克制发光、电影式留白。 |
| 进入方式 | 不展示独立启动页；在认证完成后由工作台自身按层显现。 |
| AI 导购布局 | 左侧为对话与需求线索；右侧为推荐商品。 |
| 推荐布局 | 策展主舞台：一张 AI 首选大卡，辅以两张次级推荐卡和后续列表。 |
| 全站范围 | 登录、AI 导购、商品中心、购物车、订单、偏好页统一应用新主题。 |
| 业务边界 | 仅改前端视觉与交互表现；保留所有现有接口与业务行为。 |

## 3. 设计语言

### 3.1 主题令牌

以 CSS 自定义属性定义单一主题源，所有页面和组件通过语义令牌取值，避免在组件内散落具体颜色。

| 用途 | 令牌语义 | 建议色彩 |
| --- | --- | --- |
| 页面底色 | `--bg-base` | 近黑蓝 `#080A0F` |
| 次级背景 | `--bg-elevated` | 石墨蓝 `#10131C` |
| 玻璃表面 | `--surface-glass` | 半透明蓝黑，配合 backdrop blur |
| 主要文字 | `--text-primary` | 冷白 `#F5F7FF` |
| 次要文字 | `--text-secondary` | 蓝灰 `#9AA3B8` |
| AI 强调 | `--accent-ai` | 靛蓝 `#7785FF` |
| AI 辅助高光 | `--accent-cyan` | 冰青 `#7BE7E0` |
| 价格 | `--accent-price` | 暖金 `#F2B56B` |
| 风险/错误 | `--accent-danger` | 柔和珊瑚红，保留既有语义 |
| 边框 | `--border-subtle` | 低透明冷白 |

背景以两个低透明径向光晕和细微噪点/网格营造空间感；光效不得阻碍正文、表单、图片和按钮的对比度。

### 3.2 字体、图标与层级

- 保留中文可读性，优先使用现有系统中文字体栈；英文品牌、数字与小标签用现有无衬线字体的高字重和适度字距表达科技感。
- 品牌名统一为 `NOIR.AI`，副标题保留中文产品语义，例如“智能穿搭工作台”。
- Lucide 图标继续复用；导航图标、状态图标和操作图标使用统一的 18px/20px 尺寸档位。
- 不以纯发光代替选中状态；选中导航、主按钮和匹配标签必须同时具备文字、背景和边框差异。

### 3.3 动效原则

- 默认动画短而克制：常规 hover 为 160–220ms，工作台进入为 600–900ms。
- 认证后依次呈现：背景与顶栏淡入 → 左侧 AI 面板上移淡入 → 右侧首选卡显现 → 辅助卡 60–100ms 错峰进入。
- 页面切换采用内容淡入/上移，不使用整页强闪烁、旋转或大面积位移。
- 商品卡 hover 仅允许 2–4px 上移、边框/阴影增强、图片轻微缩放；点击、加购、购买必须保留立即反馈。
- 所有新动画都由 `prefers-reduced-motion: reduce` 关闭或显著缩短。

## 4. 页面与组件设计

### 4.1 全局应用壳层

`App` 仍然负责认证分支、当前视图、购物车、AI 状态和确认弹窗。其职责不扩展为视觉状态中心。

- 顶栏改为悬浮玻璃导航：左侧品牌，中间主视图入口，右侧用户名、购物袋入口和退出操作。
- 次级功能入口（订单、偏好）在宽屏以紧凑图标/文本显示；窄屏改为横向可滚动或折叠导航，不能丢失功能。
- 全局状态信息、确认弹窗和购物车抽屉采用深色玻璃浮层，层级高于页面内容且带清晰焦点边框。
- 应用容器在认证完成后添加一次性 `is-entered` 状态，驱动分层进入动画；切换导航不重复播放完整入场动画。

### 4.2 登录页

- 保留现有登录和注册表单、校验与错误文案。
- 布局调整为居中玻璃登录面板，背后使用低亮度渐变光晕与产品标志。
- 登录按钮显示明确 loading 状态；成功后保持现有认证回调，再由应用壳层触发工作台进入动画。
- 错误文案、输入框焦点、禁用按钮符合深色主题对比度要求。

### 4.3 AI 导购工作台

AI 导购页取代当前大面积浅色 Hero，认证后直接展示工作台，首屏内可同时看到对话和 AI 推荐结果。

```text
┌──────────────────────── NOIR.AI 顶栏 ────────────────────────┐
│ 左：AI 对话 / 当前需求线索  │ 右：AI 策展推荐                 │
│                              │  ┌── AI 首选大卡 ───────────┐  │
│ 流式回复、输入框、筛选项       │  │ 商品图、匹配度、推荐理由    │  │
│                              │  │ 价格、加购、立即购买        │  │
│                              │  └─────────────────────────┘  │
│                              │  ┌─ 辅助卡 ─┐ ┌─ 辅助卡 ─┐      │
└─────────────────────────────────────────────────────────────┘
```

#### 左侧：AI 对话面板

- 承载现有 `ChatPanel`：消息、流式状态、错误、筛选项和输入均保留。
- 顶部新增仅由已有状态推导的“需求线索”模块：显示最近用户需求和现有分类/风格/季节/预算筛选；没有数据时展示等待状态，不伪造 AI 结论。
- 消息气泡按用户与 AI 区分，AI 流式状态使用细小脉冲点或光标，不新增请求。
- 输入区固定在对话面板底部，发送按钮明确可见，窄屏时仍可获得焦点与完整操作空间。

#### 右侧：AI 策展推荐区

- 推荐区继续消费 `recommendations`、`recommendationMeta` 和现有行为事件回调。
- 第一件推荐作为“AI 首选”主卡；没有 `rankScore` 时显示“AI 精选”而不是伪造百分比。
- 第二、第三件作为辅助卡；其余商品以下方响应式网格继续呈现，避免隐藏真实推荐结果。
- 主卡和辅助卡都保留图片、名称、价格、规格/库存、真实推荐理由、加购、购买和点击埋点。
- 初次加载时显示与卡片尺寸一致的骨架屏；无推荐时显示由当前空状态文案构成的暗色占位，保持继续对话的引导。
- 现有搭配清单从第三列收敛为右下角购物袋入口及购物车抽屉；AI 页仍可通过现有跳转打开购物车。该变化不移除购物车数据或结算流程。

### 4.4 商品中心

- 复用现有商品浏览数据和筛选/操作逻辑。
- 商品卡应用同一深色图片舞台、AI/库存标签、暖金价格与悬停效果。
- 筛选区改为紧凑玻璃控制栏；加载、空结果、错误状态与现有逻辑保持一致。

### 4.5 购物车、订单与偏好

- 购物车页、悬浮购物车抽屉和确认弹窗统一为深色玻璃层，突出总价、结算和取消/确认的操作层级。
- 订单页保持订单状态、支付与取消等现有行为，仅重做时间线、状态徽标和订单卡片视觉。
- 偏好页保留全部字段和保存逻辑；将分区表单改为深色网格卡片，输入与标签在高对比焦点状态下工作。

## 5. 实现结构

本轮保持现有特性目录和 API 边界，不新建跨业务状态管理层。

| 区域 | 变更责任 |
| --- | --- |
| `src/styles.css` | 定义主题令牌、基础组件、动画、响应式规则与 reduced-motion 降级。 |
| `src/app/App.tsx` | 添加仅控制一次性进入动画的展示状态与新的壳层 className；不改变认证、导航和业务状态流。 |
| `src/pages/AiShoppingPage.tsx` | 移除浅色 Hero 的主视觉职责，重组为双栏策展工作台，保留推荐加载、埋点和购物车交互。 |
| `src/features/assistant/ChatPanel.tsx` | 增加语义 className/局部结构以支撑需求线索和深色消息体验，不改变聊天请求或流式协议。 |
| `src/features/catalog/ProductCard.tsx` | 支持主卡/辅助卡等展示变体，保持现有商品数据、操作及行为事件。 |
| 其他页面和现有浮层组件 | 以语义 className 和共享主题完成视觉迁移，不修改业务请求。 |

推荐以 `ProductCard` 的展示变体实现，不复制商品业务逻辑；主卡排序只使用当前数组顺序，不重新排序、过滤或修改后端结果。

## 6. 数据流、错误与状态

### 数据流

```text
用户认证成功
  -> App 保留认证回调与既有状态初始化
  -> 应用壳层播放一次分层进入动画
  -> AiShoppingPage 拉取现有候选商品并刷新购物车
  -> ChatPanel 发送既有聊天请求并消费流式结果
  -> 现有 onRecommendations 更新 recommendation state
  -> 策展展示层按数组顺序渲染主卡、辅助卡和后续网格
  -> 现有 onAction / onBehaviorEvent 执行购买、加购和埋点
```

### 错误和边界状态

- 推荐加载失败、聊天流式失败、购物车请求失败继续使用现有错误来源和文案；视觉上以高对比提示条显示，不吞掉错误。
- 数据缺少图片时使用品牌化占位，不请求随机外部图片。
- 数据缺少 `rankScore` 或推荐理由时降级显示为“AI 精选”或商品属性，不伪造匹配度和理由。
- 低性能设备及用户启用减少动态效果时，页面保留完整可用性且不依赖动画结束事件才能交互。

## 7. 响应式、可访问性与性能

- 宽屏（约 1100px 及以上）使用左窄右宽的双栏工作台，右侧主卡可横跨两行。
- 中等宽度下对话区与推荐区保持垂直可读，辅助卡从双列退为单列或横向滚动，避免压缩商品信息。
- 窄屏（约 760px 以下）改为先对话、后推荐的单列阅读顺序；底部输入区不被浏览器安全区域遮挡。
- 所有操作必须保留文本或 `aria-label`，不能只以图标传达关键动作。
- 焦点态、正文、次要文字和标签必须满足可读对比；键盘可访问导航、输入、卡片操作、抽屉和确认弹窗。
- 只动画 `opacity` 和 `transform` 等合成友好属性；不引入大尺寸视频、远程字体依赖或重型动效库。

## 8. 验收与测试

### 自动验证

- 运行前端 TypeScript 构建，确保组件接口和样式改动不破坏编译。
- 运行现有 Vitest 单元测试，重点覆盖聊天状态、商品操作、API 客户端与偏好页。
- 运行现有 Playwright AI 导购用例，确认登录后可看到推荐、输入聊天、加购、购买和跳转购物车。

### 人工验收

- 登录后工作台在约一秒内以“顶栏 → 左侧对话 → 主卡 → 辅助卡”的顺序出现，无独立启动页，无明显卡顿。
- AI 页首屏同时可见左侧对话和右侧 AI 首选商品；现有真实商品数据、匹配度和推荐理由均未丢失。
- 商品中心、购物车、订单、偏好、抽屉和确认弹窗具有一致的深色未来主题。
- 动画关闭偏好下界面直接稳定展示，所有业务操作可完成。
- 桌面、平板和手机宽度下不存在水平溢出、不可点击操作或文字不可读问题。

## 9. 不变量

- 后端及 AI 服务仍是用户、商品、价格、库存、订单、支付和推荐结果的唯一可信来源。
- 前端不生成、不修改或伪造商品价格、库存、匹配分数、推荐理由、订单状态与支付状态。
- 业务代码与展示代码分离：视觉变体不改变 API 调用、状态归属和事件语义。
